# Projeto Git Flow
Este repositório foi criado para demonstrar o uso do Git Flow
no gerenciamento de ramificações.
## Ferramenta utilizada
A ferramenta escolhida foi o Git, com hospedagem do repositório no GitHub.
## Objetivo
Demonstrar como organizar um projeto utilizando branches específicas para:
- lançamento de versões;
- melhorias e novas funcionalidades;
- correção de bugs por meio de hotfix.
## Estrutura de branches
| Branch | Função |
|---|---|
| main | Armazena a versão oficial e estável do projeto |
| develop | Armazena o desenvolvimento principal |
| feature | Usada para criar novas funcionalidades |
| release | Usada para preparar uma nova versão |
| hotfix | Usada para correções urgentes |

## Conclusão
O Git Flow ajuda a organizar o desenvolvimento, evitando que alterações
incompletas sejam aplicadas diretamente na versão principal.
